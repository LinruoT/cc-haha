
**项目名称：** 针对中文开发环境深度优化的开源 Agent 终端

#### **1. 项目解决的核心痛点：**
目前主流 AI 编程 Agent 在中文开发语境下存在以下核心问题：
*   **指令遵循率低：** 由于原生 System Prompt 多以英文逻辑构建，国产模型在处理复杂工具调用时，常因语意理解细微偏差导致调用失败。
*   **错误经验无法沉淀：** 当 Agent 遇到内部工具（如文件 I/O、终端命令）执行失败时，往往陷入重复错误的死循环。目前缺乏有效机制将失败原因进行结构化分析，并转化为 Agent 自身的“长期记忆”。
*   **高阶工具失效后的“降级”安全风险：** 现有 Agent 在执行代码修改任务时，若自带的精密修改工具（如基于行号或 AST 的编辑工具）执行失败，模型常会退而求其次使用 `sed`、`python` 或重定向等非透明、高风险命令强行修改。这不仅导致代码格式混乱，还极易引入逻辑坏点且难以回滚。
*   **指令遵循的“黑盒”性：** 缺乏对工具执行失败后的强制纠偏机制，导致 Agent 在错误的路径上反复尝试，浪费大量 Token 却无法达成预期结果。

#### **2. 核心逻辑流：**
本项目基于 **cc-haha** 框架进行二次开发，核心逻辑包含：
*   **系统提示词中文语义对齐：** 将复杂的 Agent 调度逻辑和工具说明重构为更契合中文语意的指令集，显著降低国产大模型对工具边界理解的成本。
*   **多 Agent 协作的反思机制：** 引入长链推理逻辑。一旦主 Agent 调用工具（尤其是代码编辑工具）失败，系统将阻断其盲目使用 Shell 命令的倾向，转而触发专用的“反思 Agent”对失败日志进行归因。
*   **动态记忆闭环（Experience-Loop）：** 将分析结果持久化至“专用记忆文件”。在后续任务中，系统会根据当前上下文动态检索相关经验并注入提示词，引导模型避开已知的工具调用坑点，实现 Agent 的自适应进化。
